# 0.19.0

Deployed: 17-01-2026

Unified YAML gets first-class migration and schema tooling, plus streaming execution metrics and Kafka sink upgrades.

## Unified YAML Format & Migration

- **Migration tooling**: Automated migration script and guide for legacy plans to unified YAML
  ([Migration guide](../../migrations/yaml-unified-format/MIGRATION.md))
- **Unified YAML parsing**: Detection and conversion pipeline to load unified configs alongside legacy plans
  ([Configuration guide](../../docs/configuration.md))
- **Schema validation**: JSON schema for IDE autocomplete and validation of unified YAML configs
  ([Unified YAML schema](../../docs/configuration/unified-yaml-schema.md))
- **Configuration guidance**: Updated unified YAML configuration and examples
  ([Configuration guide](../../docs/configuration.md))

## Data Generation Helpers

- **String and pattern helpers**: `email`, `phone`, `url`, `uuidPattern`, `ipv4`, `ipv6`, `ssnPattern`, `creditCard`
- **Date/time helpers**: `withinDays`, `futureDays`, `excludeWeekends`, `businessHours`, `timeBetween`
- **Sequencing helpers**: `sequence`, `dailyBatchSequence`, `semanticVersion`
- **Conditional helpers**: `conditionalValue`, `mapping`
- **Array helpers**: `arrayWeightedOneOf`, `arrayEmptyProbability`

Check out the [Data generation helpers](../../docs/generator/data-generator.md) for more information.

## Streaming & Performance

- **Kafka streaming sink**: Streaming sink processor with buffering and timing metrics
  ([Kafka guide](../../docs/guide/data-source/messaging/kafka.md))
- **Streaming metrics**: Structured streaming performance metrics surfaced in reports and validations
  ([HTML report](../../docs/report/html-report.md), [Validation](../../docs/validation.md))

## Tooling & Examples

- **Unified YAML schema and examples**: JSON schema + sample unified configs for IDE validation
  ([Unified YAML schema](../../docs/configuration/unified-yaml-schema.md))
- **Memory profiling toolkit**: Load scenarios and scripts for profiling streaming workloads
  ([Memory profiling README](https://github.com/data-catering/data-caterer/blob/main/misc/memory-profiling/README.md))

## Migration Notes

Legacy YAML format remains supported, but unified YAML is recommended for new configurations and tooling support.
